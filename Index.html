ETAPA 1 Creación

git clone

Este comando tiene como fin de sacar un repositorio del internet
con el fin de tenerlo en nuestro propio ordenador.
debes tener Git descargado en tu ordenador

Ejecutación

git clone [URL del repositorio]

ETAPA 2 Commit

git add

Este comando prepara los archivos para ser confirmados (commit).
En otras palabras: le dice a Git “quiero que estos cambios entren en el próximo commit”.

Ejecutación

git add index.html : Añadir un archivo especifico

git add . : Añade todos los archivos cambiados en la carpeta actual

git add *.css : Añade todos los archivos que terminen .css


git commit -m "mensaje del cambio"

confirmas los cambios añadidos con git add

git commit -am "mensaje del cambio"

confirma todos los archivos modificadoso agrega archivos nuevos


Tip: Para archivos nuevos, siempre hay que hacer git add antes. Para archivos ya rastreados, -am ahorra un paso.

ETAPA 6

Comandos practicos:
  • git log:Muestra el historial de commits de tu repositorio.
    • Entre fechas
    • fecha especifica
    • Usuario especifico
    • Del día
  • git reset:Sirve para mover el HEAD (el puntero actual) a otro commit(Deshace Commit)
    • git reset --soft HEAD~1 : Deshace el último commit PERO:
      • Mantiene los cambios
      • Los deja listos para volver a hacer commit
    • git reset HEAD~1 : Deshaz el commit pero deja el archivo modificado
      • Quita los archivos del staging
      • Pero mantiene los cambios en el código
    • git reset --hard HEAD~1 : Quiero que esto nunca haya pasado
      • Borra los cambios del archivo
      • Regresa todo exactamente como estaba
  • git diff:muestra las diferencias entre tus archivos y lo que está en Git.
    • git diff: Te muestra los cambios en los archivos modificados, pero que todavía no agregaste congit add.
    • git diff --staged: Te muestra los cambios que ya agregaste con git add y que se incluirán en el próximo commit.
    • git diff commit1 commit2: Ver cambios entre commits

ETAPA 6 Las ramas

Te permiten trabajar en algo sin afectar main.

  • git branch:Muestra las ramas que existen en tu repositorio.
  • git checkout -b ""Crear y cambiar en un solo paso
  • git checkout ""Cambiamos de rama
  • Pull Request: Solicitud para que tus cambios en una rama se integren en otra rama.
    Para subir tu rama a GitHub y poder crear un Pull Request, usa:
    git push -u origin feature-mejora-estilo
    Esto crea la rama en GitHub y la conecta con tu rama local.
  • Pull Request: Solicitud para que tus cambios en una rama se integren en otra rama.
    Para subir tu rama a GitHub y poder crear un Pull Request, usa:
    git push -u origin feature-mejora-estilo
    Esto crea la rama en GitHub y la conecta con tu rama local.
    • Ve a tu repositorio en GitHub.
    • Haz clic en Pull requests → New pull request.
    • Configura:
      • Base branch: main (donde quieres unir tus cambios)
      • Compare branch: feature-mejora-estilo (tu rama con cambios)
    • Escribe un título y descripción claros de los cambios.
    • Haz clic en Create Pull Request.
  • git merge ""Unir ramas Siempre haces merge DESDE la rama destino.
